# 🤖 Python Telegram Bot with PostgreSQL

Инфраструктура для развертывания Telegram-бота и базы данных PostgreSQL в изолированных Docker-контейнерах.

---

## 🛠 Стек технологий

* **Language:** `Python 3.12`
* **Database:** `PostgreSQL 15`
* **Orchestration:** `Docker Compose`
* **Network:** `Bridge` (изолированная сеть `jsonbot_network`)

---

## 🚀 Быстрый запуск  

### 1. Настройка окружения  
Создайте файл .env в корне проекта на основе example.env.  
Токен Telegram-бота задается в переменной TELEGRAM_BOT_TOKEN.  

### 2. Сборка и запуск  
Выполните команду для сборки образа и запуска контейнеров в фоновом режиме:  

`bash
docker-compose up -d --build`  

### Примечание: После первого запуска боту требуется около 30 секунд на инициализацию.  
Скрипт load_data_async.py считывает data/videos.json и импортирует данные в базу.  
Если записи уже существуют, дубликаты не создаются.  

## 🏗 Архитектура и работа с данными  

### Преобразование запросов (Text-to-SQL)  
Проект реализует подход преобразования естественного языка в SQL-запросы.  
При получении сообщения от пользователя бот формирует запрос к LLM, которая генерирует корректный SQL-код.  
Этот код исполняется в PostgreSQL, а результат форматируется и отправляется пользователю.  

## Описание схемы и Prompt Engineering
Для взаимодействия с LLM используется системный промпт (SYSTEM_PROMPT_TEMPLATE в config/constants.py), который включает:  

Схему данных: Текстовое описание таблиц, названий колонок и типов данных (названия видео, теги, метаданные).  
Few-shot prompting: Набор готовых примеров «вопрос-ответ», которые помогают модели понять структуру ожидаемого SQL-запроса.  
Инструкции по безопасности: Ограничение на выполнение только SELECT запросов для предотвращения изменения данных.
